# Epic 87: Desktop Auto-Updater & Distribution

## Problem Statement

The Tauri desktop app needs a signed auto-update mechanism for secure distribution. Without it, users must manually download new versions, and there's no verification that binaries haven't been tampered with. This epic sets up CI-built, signed, auto-updating desktop releases.

## Current State

### Updater Plugin (Already Registered)

The updater plugin is already registered in `src-tauri/src/lib.rs`:

```rust
.plugin(tauri_plugin_updater::Builder::new().build())
```

And permissions are set in `src-tauri/capabilities/default.json`:

```json
"updater:default"
```

However, the plugin is not configured — no signing key, no endpoints, no `createUpdaterArtifacts` in the bundle config. The plugin silently does nothing without configuration.

### Tauri v2 Updater Architecture

**How it works:**
1. App calls `check()` JS API → makes GET request to configured endpoint(s)
2. Endpoint returns JSON manifest with version, platform URLs, and signatures
3. App compares manifest version with current version
4. If update available: `downloadAndInstall()` downloads the bundle, verifies Ed25519 signature, installs
5. App relaunches via `@tauri-apps/plugin-process` `relaunch()`

**Key differences from Tauri v1:**
- Signing uses Ed25519 (not RSA)
- `createUpdaterArtifacts` config controls artifact generation
- JS API is promise-based: `check()` returns `Update | null`
- Progress events: `Started`, `Progress`, `Finished`
- Custom headers supported for authenticated endpoints

### Update Endpoint URL Variables

Tauri replaces these variables in endpoint URLs at runtime:

| Variable | Example Value | Description |
|----------|---------------|-------------|
| `{{current_version}}` | `1.0.0` | App's current SemVer version |
| `{{target}}` | `darwin`, `linux`, `windows` | OS identifier |
| `{{arch}}` | `x86_64`, `aarch64` | CPU architecture |

**Platform keys in manifest:** `{OS}-{ARCH}` format:
- `darwin-x86_64`, `darwin-aarch64` (macOS Intel/Apple Silicon)
- `windows-x86_64` (Windows)
- `linux-x86_64` (Linux)

### Update Manifest JSON Format

Static JSON file served from endpoint:

```json
{
  "version": "1.2.0",
  "notes": "Bug fixes and performance improvements",
  "pub_date": "2026-03-01T10:30:00Z",
  "platforms": {
    "darwin-x86_64": {
      "signature": "<content of .tar.gz.sig file>",
      "url": "https://releases.llamenos.org/desktop/v1.2.0/llamenos_1.2.0_x64.app.tar.gz"
    },
    "darwin-aarch64": {
      "signature": "<content of .tar.gz.sig file>",
      "url": "https://releases.llamenos.org/desktop/v1.2.0/llamenos_1.2.0_aarch64.app.tar.gz"
    },
    "windows-x86_64": {
      "signature": "<content of .nsis.zip.sig file>",
      "url": "https://releases.llamenos.org/desktop/v1.2.0/llamenos_1.2.0_x64-setup.nsis.zip"
    },
    "linux-x86_64": {
      "signature": "<content of .AppImage.sig file>",
      "url": "https://releases.llamenos.org/desktop/v1.2.0/llamenos_1.2.0_amd64.AppImage"
    }
  }
}
```

**Important**: The `signature` field must be the **content** of the `.sig` file (base64-encoded Ed25519 signature), not a path or URL.

### JavaScript API

```typescript
import { check } from '@tauri-apps/plugin-updater'
import { relaunch } from '@tauri-apps/plugin-process'

const update = await check()
if (update) {
  console.log(`Update ${update.version} available (${update.date})`)
  console.log(`Notes: ${update.body}`)

  let downloaded = 0
  let contentLength = 0

  await update.downloadAndInstall((event) => {
    switch (event.event) {
      case 'Started':
        contentLength = event.data.contentLength
        break
      case 'Progress':
        downloaded += event.data.chunkLength
        // Update progress bar: (downloaded / contentLength) * 100
        break
      case 'Finished':
        break
    }
  })

  await relaunch()
}
```

**Custom headers** can be passed to `downloadAndInstall()` for authenticated endpoints:

```typescript
await update.downloadAndInstall(onEvent, {
  headers: { 'Authorization': 'Bearer token' }
})
```

### Bundle Configuration

```json
{
  "bundle": {
    "createUpdaterArtifacts": true
  }
}
```

When `true`, the build produces:
- macOS: `.app.tar.gz` + `.app.tar.gz.sig`
- Windows: `.nsis.zip` + `.nsis.zip.sig`
- Linux: `.AppImage` + `.AppImage.sig`

The `.sig` files contain Ed25519 signatures using the private key.

### Self-Hosted Considerations

Llamenos targets self-hosted deployments (VPS, Kubernetes). The update server must be simple:
- **Option 1**: Static JSON file on any HTTP server (S3, R2, Nginx, GitHub Releases)
- **Option 2**: GitHub Releases with `latest.json` artifact (generated by CI)
- **Option 3**: Self-hosted Caddy/Nginx serving static files from build artifacts

For self-hosted operators, the update endpoint URL is configured at build time. Operators can:
1. Use the project's official releases (default endpoint)
2. Mirror releases to their own server (change endpoint at build time)
3. Disable auto-updates entirely (remove updater config)

## Requirements

### Functional Requirements

1. **Update signing** — Ed25519 keypair generated, public key embedded in builds
2. **Auto-update check** — App checks for updates on launch and periodically (every 6 hours)
3. **Update dialog** — User prompted with version, release notes, progress bar
4. **CI builds** — GitHub Actions producing signed artifacts for macOS, Windows, Linux
5. **Code signing** — Apple notarization (macOS), Authenticode (Windows)
6. **Self-hosted support** — Operators can host their own update endpoint or use GitHub Releases

### Non-Functional Requirements

- Unsigned/tampered updates rejected (Ed25519 verification)
- Update check does not block app startup (background check)
- Failed update check fails silently (no error dialogs on network failure)
- Self-hosted endpoint documented with example Caddy/Nginx configs

## Technical Design

### Phase 1: Signing Key Generation

```bash
# Generate Ed25519 keypair for update signing
bunx @tauri-apps/cli signer generate -w ~/.tauri/llamenos-update-key.key

# Output:
# Public key: dW50cnVzdGVkIGNvbW1lbnQ6...
# Private key saved to: ~/.tauri/llamenos-update-key.key
# (also outputs a password for the private key)
```

**Key storage:**
- Public key → embedded in `tauri.conf.json` (committed to repo)
- Private key → GitHub Actions secret `TAURI_SIGNING_PRIVATE_KEY`
- Key password → GitHub Actions secret `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`

### Phase 2: Updater Configuration

`src-tauri/tauri.conf.json`:

```json
{
  "bundle": {
    "createUpdaterArtifacts": true
  },
  "plugins": {
    "updater": {
      "pubkey": "<Ed25519 public key from Phase 1>",
      "endpoints": [
        "https://github.com/rhonda-rodododo/llamenos/releases/latest/download/latest.json"
      ]
    }
  }
}
```

**Two endpoints** for resilience:
1. GitHub Releases (default, free hosting)
2. Self-hosted fallback (for operators who mirror releases)

### Phase 3: Update UI Component

Create a React component in the web frontend that checks for updates when running in Tauri:

```typescript
// src/client/components/UpdateChecker.tsx
import { useEffect, useState } from 'react'
import { isTauri } from '@/lib/platform'

export function UpdateChecker() {
  const [update, setUpdate] = useState<{ version: string; notes: string } | null>(null)
  const [progress, setProgress] = useState<number | null>(null)

  useEffect(() => {
    if (!isTauri()) return

    const checkUpdate = async () => {
      const { check } = await import('@tauri-apps/plugin-updater')
      const available = await check()
      if (available) {
        setUpdate({ version: available.version, notes: available.body ?? '' })
      }
    }

    // Check on mount, then every 6 hours
    checkUpdate().catch(() => {}) // Silent failure
    const interval = setInterval(() => checkUpdate().catch(() => {}), 6 * 60 * 60 * 1000)
    return () => clearInterval(interval)
  }, [])

  // ... render update banner/dialog with download progress
}
```

### Phase 4: CI Build Pipeline

`.github/workflows/tauri-release.yml`:

**Trigger**: Push to `release/desktop-*` tags or manual dispatch

**Matrix**:
- macOS: `macos-latest` (universal binary: x86_64 + aarch64)
- Windows: `windows-latest` (x86_64)
- Linux: `ubuntu-22.04` (x86_64)

**Steps per platform**:
1. Install Rust toolchain + platform deps
2. Install Bun + `bun install`
3. Build: `bunx @tauri-apps/cli build`
4. Sign: Tauri CLI uses `TAURI_SIGNING_PRIVATE_KEY` + password from env
5. Upload artifacts to GitHub Release
6. Generate `latest.json` manifest from all platform artifacts
7. Upload `latest.json` to GitHub Release

**Environment secrets required**:
- `TAURI_SIGNING_PRIVATE_KEY` — Ed25519 private key
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` — key password
- `APPLE_CERTIFICATE` — Apple Developer ID cert (p12, base64)
- `APPLE_CERTIFICATE_PASSWORD` — cert password
- `APPLE_ID` — Apple ID email
- `APPLE_PASSWORD` — App-specific password
- `APPLE_TEAM_ID` — Team identifier

### Phase 5: Code Signing

**macOS** (Apple Developer ID + notarization):
```yaml
# In CI, Tauri v2 handles notarization automatically when these env vars are set:
env:
  APPLE_CERTIFICATE: ${{ secrets.APPLE_CERTIFICATE }}
  APPLE_CERTIFICATE_PASSWORD: ${{ secrets.APPLE_CERTIFICATE_PASSWORD }}
  APPLE_SIGNING_IDENTITY: "Developer ID Application: <Team Name>"
  APPLE_ID: ${{ secrets.APPLE_ID }}
  APPLE_PASSWORD: ${{ secrets.APPLE_PASSWORD }}
  APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
```

**Windows** (Authenticode):
- Requires a code signing certificate from a CA (DigiCert, Sectigo, etc.)
- Configured via `TAURI_WINDOWS_SIGN_COMMAND` env var in CI
- Can use Azure SignTool for EV certs

**Linux**:
- AppImage is not typically code-signed
- Ed25519 signature from Tauri updater provides integrity
- Optional: GPG detached signature for manual verification

### Phase 6: Manifest Generation Script

```bash
#!/bin/bash
# scripts/generate-update-manifest.sh
# Run in CI after all platform builds complete

VERSION="${GITHUB_REF_NAME#release/desktop-v}"
BASE_URL="https://github.com/rhonda-rodododo/llamenos/releases/download/v${VERSION}"

cat > latest.json << EOF
{
  "version": "${VERSION}",
  "notes": "$(gh release view "v${VERSION}" --json body -q .body | head -500)",
  "pub_date": "$(date -u +%Y-%m-%dT%H:%M:%SZ)",
  "platforms": {
    "darwin-x86_64": {
      "signature": "$(cat artifacts/llamenos.app.tar.gz.sig)",
      "url": "${BASE_URL}/llamenos_${VERSION}_x64.app.tar.gz"
    },
    "darwin-aarch64": {
      "signature": "$(cat artifacts/llamenos.app.tar.gz.sig)",
      "url": "${BASE_URL}/llamenos_${VERSION}_aarch64.app.tar.gz"
    },
    "windows-x86_64": {
      "signature": "$(cat artifacts/llamenos_${VERSION}_x64-setup.nsis.zip.sig)",
      "url": "${BASE_URL}/llamenos_${VERSION}_x64-setup.nsis.zip"
    },
    "linux-x86_64": {
      "signature": "$(cat artifacts/llamenos_${VERSION}_amd64.AppImage.sig)",
      "url": "${BASE_URL}/llamenos_${VERSION}_amd64.AppImage"
    }
  }
}
EOF
```
