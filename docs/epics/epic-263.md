# Epic 263: Protocol & Schema Hardening

## Summary
Fix 9 protocol/schema vulnerabilities from Audit Round 8: legacy auth fallback in spec (M20), missing NotePayload maxLength (M21), hub slug pattern, blast mediaUrl constraint, i18n codegen escaping, FIELD_NAME_REGEX in schema, upload totalSize max, and tag constraints.

## Context
- **Audit Round**: 8 (March 2026)
- Protocol-level and schema-level hardening
- The protocol spec is the source of truth for all client implementations

## Implementation

### M20: Remove Legacy Auth Fallback from Protocol Spec

**`docs/protocol/PROTOCOL.md`** — remove step 7 from validation rules:
```
DELETE: "7. If bound verification fails, fall back to legacy format..."
```
The transition period is over. All clients implement method+path binding (Epic 254).

### M21: Add maxLength to NotePayload.text

**`packages/protocol/schemas/notes.json`**:
```json
"text": {
  "type": "string",
  "description": "Note text content",
  "maxLength": 65536
}
```

Server-side enforcement in `apps/worker/routes/notes.ts`.

### i18n Codegen: Escape Output

**`packages/i18n/tools/i18n-codegen.ts`**:
```typescript
function escapeIOSString(s: string): string {
  return s.replace(/\\/g, '\\\\').replace(/"/g, '\\"').replace(/\n/g, '\\n')
}

function generateIOS(keys: Record<string, string>): string {
  const lines = Object.entries(keys)
    .sort(([a], [b]) => a.localeCompare(b))
    .map(([key, value]) =>
      `"${escapeIOSString(key)}" = "${escapeIOSString(toIOSString(value))}";`)
  return lines.join('\n') + '\n'
}
```
