# Contact Import/Export & Merging — Design Spec (Revised)

**Date:** 2026-03-30 (revised from 2026-03-28 original)
**Status:** Draft
**Scope:** CSV import with client-side dedup and encryption, plaintext and encrypted export, and merge UI for resolving duplicate contacts.
**Depends on:** Contact Directory v1 (2026-03-30), Tag Management (2026-03-30), PBAC Scope Hierarchy (2026-03-30)

---

## 1. CSV Import

### Use case

Hub admin has a spreadsheet from a partner org, previous system, or rapid response event. Bulk-import into the contact directory with proper encryption and dedup.

### Flow

1. Admin navigates to `/contacts/import`
2. Uploads CSV (client-side parsing — file never sent as plaintext)
3. Column mapping UI: map CSV columns to contact fields
4. Preview first 10 rows
5. **Dedup check**: for each row with phone, client calls `POST /api/contacts/hash-phone` → `GET /api/contacts/check-duplicate`
6. Preview: N new, M potential duplicates
7. For duplicates: "Skip" / "Import Anyway" / "Merge" per row (or bulk choice)
8. Client encrypts each contact (Tier 1 + Tier 2 envelopes) and sends to `POST /api/contacts/import`
9. Progress bar
10. Summary: N created, M skipped, K errors

### Column mapping

Standard columns: Display Name, Full Name, Phone, Email, Type, Risk Level, Tags, Notes. Custom fields mapped by name/label.

### Batch API

```
POST /api/contacts/import
```

```typescript
{
  contacts: Array<{
    contactType: string
    riskLevel: string
    tags: string[]
    identifierHash?: string
    encryptedDisplayName: string
    displayNameEnvelopes: RecipientEnvelope[]
    // ... all encrypted fields
  }>
}
```

Returns: `{ created: number, errors: Array<{ index: number, error: string }> }`

Requires: `contacts:create` + `contacts:envelope-full` + `contacts:read-all` scope

### E2EE: client-side encryption

CSV parsed and encrypted entirely in browser. Server receives only encrypted blobs + plaintext metadata. Zero-knowledge preserved.

### Rate limit

500 contacts per batch.

## 2. Encrypted Export

### CSV export (plaintext)

Requires: `contacts:read-all` + `contacts:envelope-full`

1. Client decrypts all contacts
2. Generates CSV in browser
3. Triggers download

File never sent to or generated by server. Contains: Display Name, Full Name, Phone, Email, Address, DOB, Type, Risk Level, Tags, Custom Fields, Support Contacts.

### Encrypted JSON export (backup)

```typescript
interface ContactExport {
  version: 1
  hubId: string
  exportedAt: string
  exportedBy: string
  contacts: Array<{
    id: string
    contactType: string
    riskLevel: string
    tags: string[]
    encryptedDisplayName: string
    displayNameEnvelopes: RecipientEnvelope[]
    // all encrypted fields as-is
  }>
  relationships: Array<{
    encryptedPayload: string
    payloadEnvelopes: RecipientEnvelope[]
  }>
}
```

Re-importable into another instance (envelopes need re-wrapping for new instance's users).

## 3. Contact Merging

### Flow

1. From contact profile: "Merge with another contact"
2. Search/select other contact
3. Side-by-side comparison (decrypted fields)
4. For each field, pick which value to keep (or edit merged value)
5. Preview merged contact + linked data impact
6. Confirm → merge executes

### Merge logic

1. Choose primary contact (survives)
2. Re-link all calls, conversations, notes, reports from secondary → primary
3. Union tags
4. Soft-delete secondary with `mergedInto: primaryId`
5. Update primary with merge choices (re-encrypted by client)

### API

```
POST /api/contacts/:primaryId/merge
```

```typescript
{
  secondaryId: string
  encryptedDisplayName: string
  displayNameEnvelopes: RecipientEnvelope[]
  // ... merged field values (re-encrypted)
  contactType: string
  riskLevel: string
  tags: string[]
}
```

Requires: `contacts:update-all` + `contacts:envelope-full` + `contacts:delete`

### Schema addition

```typescript
mergedInto: text('merged_into')  // ID of contact this was merged into
```

## 4. Duplicate Detection

### Server-side

Show "Possible duplicates" alert in contact directory when contacts share `identifierHash` values. No decryption required.

### Future: fuzzy matching

Client-side Levenshtein distance on decrypted display names. Not in this spec.

## 6. Testing

### Unit tests
- Column mapping logic
- Dedup matching by identifierHash
- Merge field resolution (primary vs secondary)
- Tag union logic

### API tests
- Import: batch creates contacts with encrypted fields
- Import: dedup check per row
- Import: permission gating (requires `contacts:create` + `contacts:envelope-full`)
- Merge: re-links calls/conversations/notes/reports
- Merge: soft-deletes secondary with `mergedInto`
- Merge: permission gating

### UI E2E tests
- Import flow: upload CSV → column mapping → dedup preview → import
- Export: CSV download contains decrypted fields
- Merge: side-by-side comparison → field selection → merge execution
- Duplicate detection alert in directory

## 7. Non-Goals

- Import from other CRM systems — future
- Automatic merge without human review
- Cross-hub import/export
- Encrypted export re-import with automatic envelope re-wrapping — future
